# 数据定义语言（DDL）概述

数据库中，数据定义语言（Data Definition Language，简称 DDL）用于定义和管理数据库结构。DDL 主要负责创建、修改和删除数据库对象，如表、视图、索引、模式和约束。

## 主要作用

- 定义数据库结构：描述数据如何在数据库中存储。
- 管理数据库对象：创建、修改和删除数据库对象。
- 建立约束规则：维护数据完整性和一致性。

## 常见 DDL 命令

- `CREATE`：创建数据库对象，例如表、视图、索引、模式等。
- `ALTER`：修改已存在的数据库对象结构，例如增加列、修改列类型、添加约束。
- `DROP`：删除数据库对象，例如删除表或视图。
- `TRUNCATE`：快速清空表中的所有数据，但保留表结构。

## 常见数据库对象

- 表（Table）
- 视图（View）
- 索引（Index）
- 模式 / 数据库（Schema / Database）
- 序列（Sequence）
- 约束（Constraint，如主键、外键、唯一约束、检查约束）

## 典型使用场景

1. 设计和创建数据模型时定义表结构和字段类型。
2. 在系统升级或版本迭代时修改现有表结构。
3. 清理不再使用的数据库对象。
4. 为查询性能创建索引。

## DDL 与其他 SQL 语言的关系

- DDL（数据定义语言）：定义数据库结构。
- DML（数据操作语言）：对数据进行插入、更新、删除和查询。
- DCL（数据控制语言）：管理访问权限和安全性。
- TCL（事务控制语言）：控制事务的提交和回滚。

## 示例

- 创建表：
  ```sql
  CREATE TABLE employees (
      id INT PRIMARY KEY,
      name VARCHAR(100),
      hire_date DATE
  );
  ```

- 修改表：
  ```sql
  ALTER TABLE employees ADD COLUMN department VARCHAR(50);
  ```

- 删除表：
  ```sql
  DROP TABLE employees;
  ```

## 小结

DDL 是数据库结构定义与管理的核心工具，通过 DDL 命令，开发者和数据库管理员可以构建、调整和维护数据库对象，确保数据模型与业务需求一致。
